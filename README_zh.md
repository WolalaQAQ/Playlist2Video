# Playlist2Video

**语言：** [English](README.md) | 中文

Playlist2Video 是一个本地 Web UI，可以把音频文件夹转换成基于 Remotion 的歌单视频。

它会扫描本地音频文件夹，读取歌曲元数据和封面，支持用鼠标拖拽调整歌单顺序，按需在浏览器中生成预览，并在准备好后导出 1080p MP4。

## 环境要求

- Node.js 24+
- npm 11+
- FFmpeg 已安装并可通过 `PATH` 访问

## 开发运行

```bash
npm install
npm run dev
```

打开 `http://127.0.0.1:5173`。

开发终端输出已经做了可读性优化。后端日志使用对齐的 `INFO`、`WARN`、`ERROR`
标签和彩色输出，`npm run dev` 会为并行运行的 `dev:server` 和 `dev:web`
输出加前缀。交互式终端默认使用 Nerd Font 图标；如果你的终端字体无法显示，
可以设置 `PLAYLIST2VIDEO_LOG_ICONS=ascii`。支持的值包括 `auto`、`nerd`、
`ascii` 和 `none`。

## 使用流程

1. 输入包含 `.mp3`、`.flac`、`.wav`、`.m4a`、`.aac` 或 `.ogg` 的本地文件夹路径。
2. 点击 **扫描文件夹**。
3. 检查并编辑歌曲标题、歌手、时长和封面状态。
4. 拖动每首歌左侧的手柄来调整歌单顺序。
5. 在 **参数调整** 面板中调整视觉效果和导出设置。
6. 准备好后点击 **生成视频**，刷新右侧 Remotion 预览。
7. 点击 **导出 MP4** 渲染最终文件。
8. 在 `output/` 目录中找到导出的文件。

扫描、排序和参数修改不会自动重新生成预览。这样可以让大型歌单在调整项目时保持响应速度。

## 参数调整

Web UI 右侧提供常驻参数面板，设置会保存到当前项目：

- 视觉效果强度：低、中、高。
- 粒子和脉冲环效果开关。
- 导出宽度、高度、FPS、编码显示和输出文件名。
- FFmpeg 音频导出设置：编码显示、码率、采样率、声道数和音量。

修改参数后，点击 **生成视频** 才会用已保存设置重建浏览器预览。

## MVP 主题

MVP 包含一个主题：`playlist-v4`。

它包括：

- 当前歌曲封面、标题、歌手、专辑和进度。
- 右侧完整歌单面板。
- 当前播放歌曲高亮。
- 底部全宽实时频谱。
- 随节拍变化的辉光、闪光、脉冲环、粒子和封面高光。

## 多语言 Web UI

Web UI 顶部提供语言切换按钮，可在中文和英文界面之间切换。语言选择会保存在浏览器本地存储中。
